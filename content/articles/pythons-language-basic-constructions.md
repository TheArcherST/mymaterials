# Термины

Важно помнить смысл этих терминов и уметь понимать, что мы к каждому из них относим.  Зачем это нужно, станет понятно дальше.

## Языковая конструкция

Это часть текста программы, соответствующая какому-то набору правил, описанному в документации к языку программирования.  Конструкции имеют определённый смысл для интерпретатора.  Все действия которые делает интерпретатор чётко соответсвуют конструкциям в программе.

Далее мы в первом приближении рассмотрим несколько важных языковых конструкций в языке программирования Python примерно так, как они описаны в официальной документации (с некоторыми упрощениями).

## `Expression` (выражение)

Это языковая конструкция, имеющая следующие свойства: (1) её можно *вычислить*, (2) во время её вычисления могут происходить *побочные эффекты*.

Результатом успешного вычисления `expression` всегда является объект.

Простейший пример куска кода, который является корректным `expression`:

```python
"Hello"
```

Самодостаточная часть одного `expression` тоже является `expression`.  Например тут

```python
"Hello" + " world!"
```

Можно выделить 3 куска текста, под которые подойдёт термин `expression`.  При вычилсении всего `expression` сначала вычислятся два входящих в него: получаем два объекта.  Потом вычисляется результат операции между ними: получаем результат всего `expression`.

Примеры побочных эффектов, которые могут произойти при вычислении:
1. Вывод текста в консоль
2. Отправление данных по сети
3. Чтение или запись в файл
4. Изменение некоторого объекта (например, изменение списка)

Пример `expression` с побочным эффектом:

```python
print("Hello world!")
```

Обратите внимание на то, что
1. Это всё ещё expression.  Он вычисляется, и результатом вычисления будет объект `None`.  Вообще, вызов любых функций подпадает под `expression`.
2. При более детальном анализе, можно выделить внутри него ещё один `expression`.  Сначала вычилсяется результат первого (получаем объект `Hello world!`), потом результат второго (получаем объект `None`).


## `Statement` (инструкция)

Это общее название для группы языковых конструкций.  Далее перечислены некоторые из них, и для любой корректно говорить, что она является языковой конструкцией `statement`.

Примечательно, что любая программа на Python состоит из последовательности конструкций `statement`. Для разделения элементов последовательности обычно достаточно использовать перенос строки (хотя некоторые `statement` могут занимать сразу несколько строк).

Возьмём такую программу:

```python
a = 1

["Hello"]

def test():
    pass
```

В каком-то смысле можно сказать, что интерпретатор, в определённый момент, видит этот текст вот так:

```txt
statement_1 разделитель statement_2 разделитель statement_3
```

И выполнение вашего кода он понимает как последовательное выполнение этих `statement`.

Под выполнением `statement` понимается выполнение каких-то конкретных действий, заложенных в смысл каждого из них.  Далее по тексту расписан смысл некоторых видов `statement`.

### `Assignment statement` (инструкция присваивания)

Имеет вид `"name = expression"`

Смысл инструкции (интерпретатор понимает, что он должен сделать именно это при её выполнении):
1. Вычислить `expression`.
2. Cвязать результат вычисления с именем `name`.

### `Expression statement` (инструкция-выражение)

Имеет вид `"expression"`.

Смысл инструкции:

1. Вычислить выражение `expression` (и всё).

Например вот в таком коде

```python
a = [1]
[a]
```

У нас есть:
* два `statement`: `assignment statement` и `expression statement`;
* два `expression`: первое является частью `assignment statement`, второе является `expression statement`.

Получается, что на второй строчке у нас написан текст `[a]`, который является одновременно и `expression` и `expression statement`. На самом деле это означает простую вещь: к этому куску текста относится сразу два смысла:
1. Интерпретатор Python при проходе по программе выполняет этот кусок текста (смысл который мы тянем от `expression statement`)
2. Этот кусок текста вычисляется, в результате чего мы получаем объект. И в ходе вычисления могут возникнуть побочные эффекты. (смысл, который мы тянем от `expression`)

# Зачем учить все эти термины

Благодаря тому, что мы теперь знаем, как между собой связаны языковые конструкции, мы получаем возможность однообразно объяснять вроде бы разный код. Например:

```python
print("Hello")
a = 1
```

Теперь в ходе объяснения поведения первой строчки мы используем термины `statement`, `expression statement`, `expression`, а в ходе объяснения второй строчки используем `statement`, `assignment expression`, `expression`. Попробуйте в уме раскрыть смысл каждого из терминов по отношению к первой строчке и второй строчке соответсвенно (если необходимо, вернитесь к определениям).

Обратите внимание, что поменялась только одна составляющая объяснения.  Хотя раньше эти строчки могли выглядеть так, будто они делают вообще разные вещи.

Посмотрим более подробно на переиспользование тут термина `expression`. Под ним мы понимаем языковую конструкцию, которая характеризуется свойствами, описанными выше (возвращает значение, может вызвать побочный эффект).  Эта одна идея применима сразу и к куску текста `print("Hello")` на первой строчке и к куску текста `1` на второй. Нам теперь не нужно для первого вспоминать одно объяснение, а для второй другое.  Хорошо зная, что такое `expression`, мы понимаем происходящее и там и там. Тот же принцип *переиспользования смысла* относится и к другим языковым конструкциям, которые мы рассмотрели.

Польза такого подхода в дальнейшем будет становиться всё больше и больше.  Например, `expression` используется в Python повсеместно, а не только в рассмотренном примере.

Та же логика относится и к тому, почему мы учили как меняется состояние Python через рисунки.  Вместо того, чтобы запоминать, как ведёт себя код в огромном количестве ситуаций, можно выучить логику по которой работает состояние на картинке, и научиться применять эту логику в коде.
