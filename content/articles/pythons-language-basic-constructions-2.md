# Другие конструкции

Часто основными конструкциями в Python называют немного не то, что мы рассмотрели в первой части. К ним относят что-то вроде конструкций с ключевым словом `if`, `for`, `while` и так далее.  Мы же сейчас более подробно смотрим на то как устроен язык, и основными называем более абстрактные конструкции, вроде `statement` и `expression`.  Но во второй части мы рассмотрим и обычные конструкции, причём рассмотрим их так, чтобы изученное нами ранее помогало нам понимать как они работают (будем применять то самое *переиспользование смысла*, упомянутое в первой части).

Указание: возможно, после освоения первой части стоит сделать перерыв в пару дней, вспомнить её, и уже потом переходить ко второй.

## Последовательности `statement`

Посмтрим на такой код

```python
def test():
    print("Testing...")
    print("Done!")

print("Hello")
```


Как мы уже знаем, любой код на Python состоит из последовательности `statement`.  И выполнение кода означает последовательное выполнение каждого из `statement`.  В данном случае это `def statement` и `expression statement`.

Но разве...

```python,hl_lines=2-3
def test():
    print("Testing...")
    print("Done!")

print("Hello")
```

Вот это не два `expression statement`? Да, это действительно они. Правда они относятся к уже другой последовательности `statement`.

В этом коде у нас две последовательности `statement`.  Первая последовательность основная, и `statement` из неё выполняются непосредственно в рамках выполнения программы.

А вот вторая последовательность `statement` относится к содержимому `def statement`.  Эта последовательность `statement` будет выполна только в случае, если будет выполнен соответсвующий `statement` из основной последовательности.  Например так:

```python
test()
```

Так мы добавили в основную последовательность `expression statement`.  При выполнении программы мы доходим до этого `expression statement`, и в рамках его выполнения вычисляем `expression` `test()`, смысл которого как-раз и состоит в выполнении нашей второй последовательности `statement`.

Теперь давайте введём термин для языковой конструкции, которая соответсвует вот этими самым перечислениям `statement`.  Просто чтобы не писать всегда `перечисление statement`.

И тут есть нюанс.  Если мы посмотрим в официальную документацию Python, мы увидим, что для обозначения перечислений `statement` всей программы используется термин `file_input`, а для других перечислений `statement` используется термин `suite`.

Почему не назвать и то и то одинаково, например просто `statement_list`, и использовать его и для всего файла и для отдельных последовательностей? Дело в том, что есть некоторые отличия между последовательностью во всём файле и отдельной последовательностью.  Но они нам сейчас не важны.  Поэтому мы будем использовать два термина, но и первый и второй будет означать любую последовательность языковых конструкций `statement`.

Рассмотрим вот такой код

```python
for i in range(100):
    print("Ok")
    if i % 2 == 0:
        print(i)
```

Вопрос: сколько тут последовательностей `statement`?  Правильный ответ такой: тут их 3 штуки.

* Первая `file_input`, именно она просто последовательно выполняется при запуске программы.  И в ней есть только `for statement`, всё.
* Вторая `suite`, которая является частью `for statement`, содержит два `statement`: `expression statement` и `if statement`.
* Третья `suite`, которая является частью `if statement`, содержит один `statement`: `expression statement`.

Далее будут в упрощённом виде изложены некоторые конструкции, использующие `suite`.

## `def statement`

```
"def" name "(" parameters ")" ":" suite
```

Кавычки используются чтобы обозначить, что буквально этот текст используется при записи конструкции.  Т.е. вы не можете написать `def statement` без использования текста `def`, и так далее.  Когда кавычек нет, имеется ввиду ссылка на какую-то другую конструкцию.  Под `name` тут имеется ввиду любое корректное имя, под `parameters` имеется ввиду любой корректный список параметров функции, под `suite` имеется ввиду любая корректная последоватльность `statement`.  Мы не будем сейчас подробно рассматривать некоторые конструкции вроде `arguments`.

Смысл выполнения этого `statement`:
1. Создать объект типа `function`, который содержит в себе данную последовательность `statement` и информацию об аргументах, указанную в `arguments`.
2. Связать этот объект с именем `name`.

## `if statement`

```
"if" expression ":" suite
["elif" expression ":" suite]
["else" ":" suite]
```

Квадратные скобки используются, чтобы указать, что эта часть языковой конструкции необязательна.  Вот пример в котором спользованы только обязательные части


```python
if 1 == 1:
    print("Hello")
```

Вот пример где дополнительно использована одна необязательная часть

```python
if 1 == 1:
    print("Hello")
else:
    print("Not hello")
```

Смысл выполнения этого `statement`:
1. Если результат вычисления первого `expression` это объект, который считается истиной (например объект `True`), то выполняем первый `suite`
2. Если результат первого `expression` оказался ложным, и у нас есть блок `elif`, то вычисляем следующий `expression` и если он истинен, выполняем `suite`.
3. Если результаты всех `expression` оказались ложными, и у нас есть блок `else`, выполняем последний `suite`.

Примечание: блоков `elif` может быть много.

## `while statement`

```
"while" expression ":" suite
```

Смысл выполнения этого `statement`:
1. Вычисляем `expression`.  Если результат вычисления истинен, выполняем `suite` 
2. Вычисляем `expression`.  Если результат вычисления истинен, выполняем `suite` 
3. И так далее.  Если в какой-то момент `expression` оказался ложным, выполнение `while statement` заканчивается.

## `for statement`

```
"for" name "in" expression ":" suite
```

Смысл выполнения этого `statement`:

1. Вычисляем `expression`.
2. Связываем имя `name` с первым элементом результата вычисления (если в результае вычисления в принципе элементов не может быть, например у него тип `int`, то произойдёт ошибка).
3. Выполняем `suite`.
5. Связываем имя `name` со вторым элементом результата вычисления. 
6. Выполняем `suite`.
7. И так далее, пока элементы не закончатся.


Примечание: если в результате вычисления `expression` получился объект у которого в принципе могут быть элементы, но их количество равно нулю, то наш `suite` никогда не будет выполнен.
